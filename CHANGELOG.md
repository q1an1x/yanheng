# 更新记录

## 0.3.0 - 2026-07-17

- HTTP 改用纯言序 `yanxu-request`，日期与 1min 时间范围改用纯言序 `yanxu-datetime`；UI 库不再承载业务网络与时钟能力。
- 保存设置与连接测试拆分，保存完成回调不再重启定时器并触发整套刷新，修复保存后卡顿崩溃。
- HTTP 错误保留状态码及 sub2api `code/message`；401 会显示本次实际发送 KEY 的长度与末四位，便于核对实例和凭据。
- 安全输入框恢复右键粘贴与 `Command-V`，避免界面显示的 KEY 未实际更新。
- 最低言序版本提升至 1.1.12，采用官方大整数 YXB 精确往返修复，保证 `yanxu-datetime` 可被可靠打包加载。

## 0.2.2 - 2026-07-17

- 修复启动期间嵌套回调被误判为投递失败，导致升级到 0.2.1 后菜单栏项目不出现的问题。

## 0.2.1 - 2026-07-17

- 实时信息新增真实 1min 窗口，使用显式起止时间避免 sub2api 将未知 `time_range` 回退为 1h。
- 修复“连接与预警”展开后表单超出菜单弹窗、设置值不可见且无法调整的问题。
- 修复点击“退出”后应用进程仍驻留的问题。

## 0.2.0 - 2026-07-17

- 删除言衡应用专属 Swift 与伴生动态库，API、配置、定时与统计全部改为纯言序实现。
- 修复应用图标白色方形背景，发布 PNG 现在具有真实透明边缘。
- 修复数值 Stepper 不显示当前值，并将默认最少可用账号改为 1。
- 从账号 `extra` 正确读取 Codex 与 Anthropic 的 5h/7d 用量窗口。
- 新增今天、7 天、总计周期切换，以及模型分布、用户消费榜、请求明细、SLA 和实时 QPS/TPS。
- Token 输入、输出、缓存读写、请求、消费和总量均使用独立 SF Symbol 图标。

## 0.1.0 - 2026-07-17

- 首个公开版本。
- 原生 macOS 菜单栏应用，基于 `yanxu-macos-ui`。
- 聚合 sub2api 账号可调度状态、5h/7d 平均余量与补充预警。
- 展示今日输入、输出、缓存读写 token，以及 RPM、TPM、请求和成本。
- 管理员 API Key 存入 macOS Keychain，支持自动与手动刷新。
